### Cycle 1

**Approach**: Bundled all 7 CHANGES into a single implementation pass since they share one acceptance criterion (the rebrand) and span 8 independent source files. Followed BA's affected-file list verbatim — no exploratory grep beyond the BA-prescribed scope.

**Rationale per change**:

1. **CHANGE 1 (rebrand + bigger centered header)**: Renamed all 4 in-app source occurrences (`app/page.tsx:14` h1, `app/layout.tsx:8` metadata title, `lib/prompts.ts:3` comment, `lib/prompts.ts:6` system prompt opening). For the "FeedbackForge Coach" → rename, dropped the redundant "Coach" suffix per BA AC1 implementation note (user intent is brand renaming, not introducing redundancy). For the header sizing/layout, used Tailwind classes `text-4xl font-bold` (per spec verbatim text-4xl), `text-center` on the header element, and `mb-12 py-8 sm:mb-16 sm:py-10` for the increased padding/margin (visibly larger than the baseline `mb-8 sm:mb-10`). Switched the header inner div from `flex items-center gap-2` to `inline-flex items-center justify-center gap-2` so the Sparkles+title pair renders centered as a unit while preserving the icon next to the wordmark.

2. **CHANGE 2 (NC logo)**: Inline-flex 32px circular div using `flex h-8 w-8 items-center justify-center rounded-full bg-blue-600 text-xs font-bold text-white`, placed immediately to the LEFT of the company-team text inside an `inline-flex items-center justify-center gap-2` parent. Added `aria-label="NormalCompany"` on the circle (BA C1 — small a11y win, no scope expansion). The circle wraps a literal `NC` text node so it renders correctly even with screen readers (the aria-label overrides for AT users).

3. **CHANGE 3 (tenure + Briefcase icon)**:
   - 3a: Updated all 4 tenure strings in `lib/employees.ts` to `"<N> year[s] at NormalCompany"` verbatim per spec lines 136-139. Casing and "at NormalCompany" suffix preserved exactly.
   - 3b: Edited `components/EmployeeCard.tsx` rather than `app/page.tsx` per BA implementation note (Section 1 line 26 / BA spec line 63). Replaced `<Badge variant="secondary">{employee.tenure}</Badge>` with an inline-flex span containing the lucide `Briefcase` icon (`h-4 w-4 text-slate-500`) followed by plain tenure text (`text-sm text-slate-500`). The `age` Badge remains unchanged (BA pre_existing_guard L159). Added `Briefcase` to a new `lucide-react` import line — kept on its own line per existing import style.

4. **CHANGE 4 (remove Start Over)**: Edited `components/DraftActions.tsx` per BA implementation note (BA spec line 65). Deleted the third `<Button>` block (L30-36 in baseline) and stopped destructuring `onStartOver` in the function body. Per BA S1 recommendation, kept `onStartOver` in the `Props` type so the existing call site in `components/ChatTranscript.tsx` (which spreads `{...draftActions}`) still typechecks. This is the smallest possible diff — caller code is untouched.

5. **CHANGE 5 (clipboard + new toast + 3000ms)**: Two-file edit per BA M6:
   - `lib/use-chat-page-actions.ts`: Extended the `Args` type with `messages: Message[]`. Added an internal `getLastAssistantContent()` helper that walks the array from the tail to find the last `role === 'assistant'` message (defensive — guards against the unlikely "trailing user message" case). Wrapped `navigator.clipboard?.writeText()` in a try/catch (BA risk note: insecure context, missing user gesture, denied permission). The toast string is extracted to a `const ACCEPT_TOAST` so it's verbatim from spec line 156-157 and easy to verify by grep. Delay bumped from 2000ms to 3000ms per spec line 159.
   - `components/ChatPageBody.tsx`: Passed `session.messages` into the hook call site (one-line addition). No other change.

6. **CHANGE 6 (META-RULES block)**: Inserted the verbatim 10-line META-RULES block from spec Section 5 lines 165-176 into `lib/prompts.ts`, with a blank line above it (separating from SPECIAL CASES) and a blank line below it (separating from CONSTRAINTS). Placement is immediately above the `CONSTRAINTS:` line (now L117 post-CHANGE-7-expansion), per spec line 162.

7. **CHANGE 7 (harassment redirect-to-HR)**: Replaced the existing 2-line `- Harassment/discrimination/safety: Do NOT draft feedback. Flag for HR escalation.` (baseline L91-92) with the verbatim 10-line block from spec Section 5 lines 183-193. The block stays inside the SPECIAL CASES section between the "Nothing to say" bullet and the "Emotionally charged venting" bullet — no structural reordering of the section.

**Rationale for header function extraction**: The quality-gate hook rejected the inline `Home` function when it grew to 33 lines (max 30). Extracted the header markup into a small `DashboardHeader()` component within the same file — keeps the diff minimal (one new local function, no new exports) and resolves the gate without touching unrelated code.
