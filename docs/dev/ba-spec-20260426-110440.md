## Edge Cases & Risks

- **CHANGE 5 risk**: `navigator.clipboard.writeText` is async and can reject (insecure context, denied permission, or user gesture missing). Dev should `void`-fire it (or `.catch(() => {})`) but must not block the toast/navigation flow on its rejection. The user did not specify a fallback path, so silent best-effort is acceptable — toast still shows, navigation still happens after 3s.
- **CHANGE 5 edge case**: If `session.messages` is empty or the last message is from `user` (shouldn't happen in `drafted` state per `lastIsDraft` guard, but defensively), `writeText` should be skipped — but DraftActions only renders when `lastIsDraft(...)` is true, so the guard already exists. Dev may still want a defensive `?.content ?? ""`.
- **CHANGE 1 risk**: The `<header>` already has `mb-8 sm:mb-10`. Centering via `text-center` will affect the sub-paragraphs too — including the `Logged in as Alex...` line. User said "Center the entire header section" so this is intended; dev should center all three children (h1 + two `<p>`).
- **CHANGE 2 risk**: Putting the NC div + the company text inline with `gap-2` will require restructuring the current `<p className="mt-1 text-sm text-slate-500">NormalCompany — Marketing Team</p>` into a flex container. User did not say to keep `<p>` semantics; an `<div className="inline-flex items-center gap-2">` wrapper containing the circle + a `<span>` is acceptable.
- **CHANGE 6 placement**: "near the end of the system prompt, before the CONSTRAINTS block" — `CONSTRAINTS:` appears at L98. The META-RULES block must be inserted BETWEEN the closing of SPECIAL CASES (L96, currently `Apply SBI...add "How can we get more of this?"`) and `CONSTRAINTS:` (L98), with a blank line on each side. After CHANGE 7 expands the harassment line, line numbers will shift — this is OK.
- **CHANGE 7 placement**: Replace lines 91-92 with the new multi-line block. The new block contains "End your message as a CONCLUSION" — it terminates the conversation, but the prompt's PHASE 1/2/3 + OUTPUT FORMAT machinery remains untouched. The model's behavior in this branch is harassment-specific, so no risk to other branches.
- **No risk on CHANGE 3**: Tenure string is consumed by `lib/prompts.ts:16` (`Tenure: {employee.tenure}`) and `components/ChatTopbar` (`tenure={employee.tenure}`). Both happily accept the longer string. The prompt's tenure-calibration logic uses age-based thresholds anyway (the Junior/Mid/Senior bullets), so the longer tenure string does not alter behavior.
- **TypeScript risk**: Extending `useChatPageActions` to accept `messages: Message[]` requires updating the call site in `ChatPageBody.tsx`. If dev forgets, TypeScript build fails — caught by `npm run build` in QA.
